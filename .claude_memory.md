## 🏗️ ARCHITECTURE FRONTEND (Assets/Scripts/DiscordBridge/)
L'architecture C# suit un pattern MVVM strict sans exceptions réseau (`ApiResult`)[cite: 3].
- **Networking :** `DiscordAPIBridge.cs` (MonoBehaviour Singleton)[cite: 3].
- **Data (ScriptableObjects) :** `PlayerProfileData`, `InventoryData`, `ItemDatabase`[cite: 3].
- **Session :** `SessionStore.cs` persiste le `discord_id` via PlayerPrefs[cite: 3].
- **Controllers :** `LinkAccountController`, `ProfileSyncController`, `WaveRewardController`[cite: 3].
- **Règle absolue :** L'UI ne connaît jamais le Networking ni les DTOs, elle écoute uniquement les ScriptableObjects via des events (`Action`)[cite: 3].

## 📡 BLUEPRINT BACKEND (CONTRAT D'INTERFACE REST)
- **Réseau :** Polling HTTP/JSON pur (aucun WebSocket). Mono-process uvicorn. Exposé via Cloudflare Tunnel.
- **Authentification :** Header `X-Api-Key` requis partout. Identifiant unique = `discord_id` (string).
- **Routes :**
  - `POST /api/link-account` : Échange du PIN contre un `discord_id`.
  - `GET /api/player/{discord_id}` : Route de polling principale (Mana, préférences).
  - `GET /api/inventory/{discord_id}` : Récupère les items possédés.
  - `POST /webhook/game-event` : Unity alerte d'une attaque Discord → **Re-poller `/api/player` ensuite.**
  - `POST /api/add-mana` : Récompense de vague. **Route signée HMAC**.
  - `POST /api/consume-item` : Consomme un objet (FIFO).

## ⚠️ PIÈGES RÉSOLUS (ANTI-RÉGRESSION)
- **Newtonsoft.Json :** Requis (installé via `manifest.json`) pour la compilation des DTOs[cite: 3].
- **Signature HMAC (`/api/add-mana`) :** Le timestamp calculé côté C# DOIT avoir une décimale `.0` ajoutée manuellement pour correspondre au formatage float natif de Python côté serveur, sinon la signature échoue[cite: 3].
- **Type du `discord_id` :** Renvoyé en `long` (nombre JSON) sur la route `/link-account`, mais traité comme `string` sur les requêtes `/player` et `/inventory`. Toujours le manipuler en `string` dans Unity[cite: 3].
- **Asynchronisme :** Utilisation exclusive de la nouvelle API Unity 6 `Awaitable` (pas de `Task`) pour garantir le retour sécurisé sur le thread principal[cite: 3].
